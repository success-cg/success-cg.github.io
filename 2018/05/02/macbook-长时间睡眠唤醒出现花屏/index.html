macbook 长时间睡眠唤醒出现花屏

macbook 长时间睡眠唤醒出现花屏，修改 pmset 睡眠参数方法

本人是 macbook pro touchbar 2017 款，入手半年多了，更新系统到 high sierra，会出现一个状况

长时间睡眠唤醒后，先白色花屏一秒，然后再出现输入密码界面

说实话，这个情况对使用完全没有影响，一般也就隔夜，或者隔上一个小假期，再去打开会出现白色花屏闪一下。不过，就是感觉不爽！ 想象一下，隔那么久去“临幸”一下你的MBP，结果它居然对你甩一秒脸子……

我在网上查了很久，网上各路大神的方法总结下来大概是这么几个

  • 重置 Mac 上的系统管理控制器 (SMC)
  • 重置 Mac 上的 NVRAM 或 PRAM

具体方法：

重置 Mac 上的系统管理控制器 (SMC)

  1. 选取苹果菜单 >“关机”。
  2. 等 Mac 关机后，按下内建键盘左侧的 Shift-Control-Option，然后同时按下电源按钮。按住这些按键和电源按钮 10 秒钟。
  3. 如果您的 MacBook Pro 带有触控 ID，则触控 ID 按钮也是电源按钮。
  4. 松开所有按键。
  5. 再次按下电源按钮以开启 Mac。

重置 Mac 上的 NVRAM 或 PRAM

  1. 关机，然后开机并立即同时按住以下四个按键：Option、Command、P 和 R。大约 20 秒后松开这些按键，在此期间看似在重新启动。
  2. 在发出启动声的 Mac 电脑上，您可以在两次启动声之后松开这些按键。

效果

说实话，尝试之后，基本都不管用，长时间不去“临幸”MacBook仍然会开盖白色花屏一秒的现象。

进一步研究

h折腾了好久，就是不爽，治不了你！

首先观察现象，MacBook 长时间睡眠，开盖唤醒的速度特别慢，要好几秒才出现登录界面，如果运气不好，就可能出现白色花屏。但是，一般日常使用时候，短时间合上盖子睡眠，就唤醒贼快，完全不会有花屏现象。说明长时间睡眠和短时间睡眠的存储方式是不一样的。

  • 短时间睡眠数据是存在内存中

  • 长时间睡眠数据是写到SSD中

OSX 的睡眠参数，可以打 pmset -g 了解一下你的电脑处在什么睡眠模式下：

hibernatemode

  • hibernatemode 0：睡眠时内存保持供电，其他部件断电，内存数据不写入硬盘。优点是可以快速唤醒、快速睡眠，因为内存保持供电，无论是唤醒还是进入睡眠都是1秒内的事情；节省SSD空间和寿命，因为内存数据不写入硬盘，所以硬盘里不会有内存镜像，也不会缩短SSD寿命。但是如果在睡眠的过程中电池没电的话，Mac会丢失所有内存种的数据，数据安全性不是很高。
  • hibernatemode 25：只能通过pmset设置。 系统会将内存副本存储到永久性存储器（磁盘）中，并将删除内存的电源。 系统将从磁盘映像中恢复。 如果您想要“休眠” - 睡眠较慢，唤醒较慢以及电池使用寿命较长，则应使用此设置。
  • hibernatemode 3：0和25的结合，睡眠时内存数据写入硬盘，同时保持内存供电。这个模式也叫Safe Sleep，是Mac的默认睡眠模式。

autopoweroff:

ao如果Mac处于睡眠状态经过指定的时间后，自动把内存数据写入硬盘，并切断所有部件的电源，进入休眠状态，1是开启 0是关闭。

StandBy

是在基于SSD硬盘下对Hibernate的一种改进。StandBy将RAM保存到非易失性存储后仍对至少对键盘供电，按任意键可以唤醒。不过仅在hibernatemode为3的时候起作用，1是开启 0是关闭。

修改 pmset 参数调整mac睡眠方式

这样 MacBook 合上盖子后，数据就存储在内存中了

好处

  • 翻盖唤醒快，秒唤醒，因为存在内存中
  • 减少写入 SSD 的次数，延长 SSD 使用寿命
  • 避免长时间待机，唤醒花屏的情况（本人的目的）

坏处

  • 电池续航略微变短，因为要持续给内存供电（感觉没什么关系，充电呗）
  • 如果断电，内存中的数据会丢失，对于数据安全要求较高的情况，慎用（个人用完全没关系啦）
